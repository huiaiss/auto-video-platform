## 一、项目定位

**制造业实体商家的 AI 视频工厂。** 目标客户是工厂老板、设备厂家、实体门店。

### 一句话

给设备拍段视频 → AI 看懂是什么机器、亮点在哪里 → 10 分钟出宣传片 → 直接发抖音/快手。

### 已覆盖的视频类型

| 类型 | 场景 | 状态 |
|------|------|------|
| **企业/工厂宣传** | 真实产线+产品，展现制造实力 | ✅ 隆江绕线机验证中 |
| **AI破绽鉴定**（AI照妖镜） | 科普类，AI生成内容识别教学 | ✅ 首个测试用例跑通 |
| **产品展示** | 单品多角度展示+参数讲解 | 🔄 开发中 |
| **客户案例** | 客户工厂实拍+使用效果+证言 | 📋 规划中 |
| **技术科普** | 行业知识+技术原理讲解 | 📋 规划中 |

### 目标客户

| 客户类型 | 典型需求 | 痛点 |
|---------|---------|------|
| 自动化设备厂 | 展示机器运行、精度、产能 | 机器在动，不知道拍哪个角度 |
| 工厂/代工厂 | 展示产线规模、品控能力 | 想接外贸单，没有像样视频 |
| 实体门店 | 展示手艺、设备、案例 | 只会用抖音拍15秒 |
| 外贸/跨境电商 | 产品多角度展示+参数+证书 | 国外客户要视频，找人拍太贵 |

## 二、系统总架构（三层叠加）

整个系统由 **三层架构** 叠加组成，越往下越接近代码，越往上越接近决策：

```
┌─────────────────────────────────────────────────────────────────────┐
│  L3: 12-Agent 协同工作室（规划中）                                    │
│  总导演 → 12个专业Agent 并行协作，互相评审，退回重做                    │
│  ─── 大脑层：做决策、审质量、调度资源                                │
├─────────────────────────────────────────────────────────────────────┤
│  L2: 3大引擎（设计的完整能力）                                         │
│  引擎A 创意大脑 → 引擎B 内容工厂 → 引擎C 出图+出片                     │
│  ─── 逻辑层：素材→创意→方案→成品                                    │
├─────────────────────────────────────────────────────────────────────┤
│  L1: 6层管线 + 3 Stage（当前实际运行）                                │
│  go.py → pipeline.py → 3个Stage → 3道审计 → final.mp4               │
│  ─── 执行层：7分钟跑完一条片子                                      │
└─────────────────────────────────────────────────────────────────────┘
```

│  AssetAnalyzer (素材分析层) — 当前简化版                         │

│  （设计原版9检测器，当前只用核心2个）                            │
├──────────────────────────────────────────────────────────────────┤
│  ScriptGenerator (方案生成层)                                    │
│  抽帧 → 豆包Vision → DeepSeek → script.json                     │
├──────────────────────────────────────────────────────────────────┤
│  CompositionBuilder (视频合成层)                                 │
│  TTS → 字幕 → BGM → 音效 → 品牌尾帧 → ffmpeg编码               │
├──────────────────────────────────────────────────────────────────┤
│  QualityGate (质量门控层) — 当前3道审计                          │

├──────────────────────────────────────────────────────────────────┤
│  PublishingLayer (发布策略层) — 未实现                           │

### 3 Stage 管线流程

```
go.py (一键入口)
  │
  ├── Stage 0: 素材诊断 (30s)
  │   ffprobe 扫描 → 编码/分辨率/时长/HDR报告
  │
  ├── Stage 1: 视频粗剪 (~2min)
  │   1. HDR→SDR (reinhard tonemap)
  │   2. scene filter > 0.3 场景切分
  │   3. Laplacian 清晰度40% + 曝光30% + 构图30% 评分选段
  │   4. xfade 0.3s 交叉溶解拼接
  │   5. 输出 roughcut.mp4 (静音, ~33s)
  │   │
  │   └── audit_roughcut.py 5/5 PASS
  │       (黑帧/冻结/时长/抽帧/场景数)
  │
  ├── Stage 2: 脚本生成 (~2min)
  │   1. 从 roughcut 均匀抽 6 帧
  │   2. 豆包 Seed 1.6 Vision 逐帧分析画面
  │   3. DeepSeek 根据帧描述 + 品牌数据 生成 script.json
  │   4. 输出 script.json
  │   │
  │   └── audit_script.py 5/5 PASS
  │       (品牌/字数/emotion/尾帧/时长对齐)
  │
  └── Stage 3: 视频合成 (~3min)
      [3.0] TTS 口播 (Edge TTS / 豆包 Ark TTS)
      [3.1] drawtext 字幕叠加 (底部居中, 黑底白字)
      [3.2] BGM 混音 (14首曲库, EQ滤波, 尾淡出2s)
      [3.2a] SFX 音效 (6种, 按animation类型匹配, adelay对齐)
      [3.3] 品牌尾帧 (品牌色底 + 公司名 + slogan)
      [3.4] concat + libx264 CRF20 → final.mp4 (36s, 8-9MB)
      │
      └── audit_compose.py 5/5 PASS
          (编码/大小/花屏/文字/品牌尾帧)
```

## 七、反同质化设计体系

**核心原则**：模板是骨架、Brand DNA决定视觉身份、叙事角度决定内容方向、素材驱动随机变异。

### 三刀砍同质化

#### 第一刀：Brand DNA → 视觉上就不同

```yaml
# 隆江绕线机
brand_dna:
  color_primary: "#0057b8"     # 品牌蓝
  font_primary: "思源黑体 Bold"
  logo_position: "bottom-right"
  transition_style: "hard_cut"
  narration_style: "专业稳重"
```

#### 第二刀：8种叙事角度 → 同一台机器讲8个故事

| 角度 | 结构 | 适用 |
|------|------|------|
| 痛点式 | pain_hook→solution→feature→result→cta | 抖音获客 |
| 技术硬核 | precision_hook→process→patent→spec→cta | 行业影响力 |
| 客户证言 | customer_context→pain→solution→result→cta | B2B转化 |
| 产线直击 | factory→material→machine→qc→product→cta | 工厂展示 |
| 参数对比 | old_way→new_way→spec→efficiency→cta | 采购决策 |
| 创始人视角 | philosophy→insight→principle→showcase→cta | 品牌IP |
| 慢工细活 | visual_hook→slow_motion→detail→wide→brand | 高端形象 |
| 快节奏爆款 | claim→proof→spec→result→cta | 信息流投放 |

#### 第三刀：素材驱动变异

不是人选模板，是AI看过素材后主动决策：
```
素材上传 → AI分析输出高亮画面
  → 高亮画面类型 → 推荐匹配角度
  → 品牌预设偏好 → 过滤不合适的
  → 发布平台 → 角度适配
  → 随机变量 → 防同质化最后一道防线
```

### 变异因子

| 因子 | 数量 | 说明 |
|------|------|------|
| 叙事角度 | 8种 | 不同结构+语调 |
| BGM曲库 | 14首 | 同风格内随机 |
| 语速 | 3档 | 0.9x/1.0x/1.1x |
| 色调方案 | 5套 | 品牌调色变体 |
| 字幕样式 | 4种 | 位置/大小/颜色 |
| 节奏 | 3种 | 快切/匀速/慢镜 |

**理论组合：8 × 14 × 3 × 5 × 4 × 3 = 20,160+ 种，几乎不可能撞车。**

## 十、铁律

1. **视觉只用豆包** — DeepSeek 不支持图片，千问不准，锁死
2. **脚本必须基于画面** — 先抽帧→看图→写脚本，不套模板
3. **每步必须审计** — 5/5 PASS 才进下一步，失败退回最多 3 次
4. **不同设备不同脚本** — 绕线机讲精度，空压机讲气压，水泵讲扬程
5. **素材分析永远在前** — 不预设脚本，一切由素材实际分析结果驱动
6. **Brand DNA 锁定** — 同企业所有视频自动统一风格
7. **真实素材优先** — 企业实拍，AI 润色，不虚构
8. **先提方案再动手** — 任何改动前先汇报方案，等确认再干
9. **产出不自审等于没做** — 磁盘对账 + 内容审计过了再报完成
