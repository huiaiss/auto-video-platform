# Auto 闭环管道方案 v1

> Hermes 撰写 | 2026-06-04 | 等 Claude 修完 P0 后开工

---

## 总览

当前 auto 管线只到"出片"。完整闭环需补齐三步：

```
素材 → 视频 → 发布 → 被搜到 → 客户点进来 → 留资 → 成交
 ←── Stage 1-3 ──→ ←── 本文覆盖 ──→ ←─ 客户CRM ─→
```

---

## 第一步：四平台差异化发布层

### 目标
一条片子输出四套发布物料，每条物料适配目标平台的算法和用户习惯。

### 各平台差异

| 平台 | 分辨率 | 时长 | 风格 | 标签策略 |
|------|--------|------|------|---------|
| 抖音 | 1080×1920 竖版 | 25-40s | 视觉钩子前三秒 + 痛点 + 方案 + CTA | 5-8个行业热词 |
| 快手 | 1080×1920 竖版 | 20-35s | 更接地气、方言可选、价格强调 | 3-5个本地化标签 |
| 视频号 | 1920×1080 横版 | 40-60s | 技术参数 + 工程师IP + 品质可信度 | 3-5个精准行业词 |
| 小红书 | 1080×1440 竖版(3:4) | 25-35s | 高颜值画面 + 经验分享口吻 | 2-3个品类标签+场景描述 |

### 技术实现

Pipeline 新增 Stage 4 `publish_layer.py`：

```
Stage 3 输出 → Stage 4 发布层
  ├─ 横版原片 (1920×1080) → 视频号版本
  ├─ 竖版裁剪 (1080×1920) → 抖音版本
  │    ├─ center-crop 核心画面
  │    └─ 可选：关键帧放大 + 动态跟踪（暂不实现）
  ├─ 竖版裁剪 (1080×1920) → 快手版本（复用抖音裁剪）
  └─ 3:4 裁剪 (1080×1440) → 小红书版本
```

每套物料输出：

```
output/publish/{设备ID}_{日期}/
  ├── douyin.mp4
  ├── douyin_metadata.json       # 标题 + 标签 + 发布时间
  ├── kuaishou.mp4
  ├── kuaishou_metadata.json
  ├── shipinhao.mp4
  ├── shipinhao_metadata.json
  ├── xiaohongshu.mp4
  └── xiaohongshu_metadata.json
```

### metadata.json 格式

```json
{
  "platform": "douyin",
  "title": "一台机器搞定无刷电机绕线｜台州隆江LJ-800",
  "title_alt": ["绕线效率提升3倍的秘密", "无刷电机绕线机选购避坑"],
  "tags": ["#无刷电机", "#绕线机", "#自动化设备", "#台州工厂", "#电机生产"],
  "description": "LJ-800 绕线速度3000rpm，线径0.1-1.2mm全兼容。看实拍效果。",
  "best_publish_time": "工作日 11:30-13:00 或 20:00-22:00",
  "cover_frame_time": 1.5
}
```

### 依赖
- `ffmpeg` crop filter（已有）
- LLM 生成标题/标签（Stage 2 扩展一个 prompt）
- 耗时：约 2 天（Claude 编码）

## 第二步：GEO 结构化数据（让 AI 搜得到）

### 为什么重要

2026 年采购决策路径变了：
- 以前：百度搜 → 点官网 → 打电话
- 现在：DeepSeek/豆包/元宝问"无刷电机绕线机哪家好" → AI 推荐 → 直接联系

AI 搜索引擎不看关键词密度，看**结构化信息**。追马网做了个实验：
- 同等内容，加结构化 JSON 的页面被 AI 推荐的概率高 **3.2 倍**
- 87% 行业长尾词覆盖，3 个月自然流量 70%

### 技术实现

**Stage 2 扩展**：在生成脚本的同时，从 Brand DNA 配置里提取数据，生成结构化 JSON。

1. `product-geo.json` 放在客户子域名根目录
2. HTML 落地页的 `<head>` 区嵌入 JSON-LD（搜索引擎标准格式）：

```html

  "@context": "https://schema.org",

```

### 依赖

- Stage 2 加一个 LLM prompt 生成 GEO 数据
- 落地页模板嵌入 JSON-LD（第三步）
- 耗时：约 1 天

## 第三步：设备专属落地页

### 为什么重要

工厂买家是理性决策者。他们看完视频要看三样东西才联系：
1. **参数**——跟现有产线能不能对接
2. **案例**——同行有没有在用
3. **资质**——认证/工厂实拍/验厂视频

如果视频看完跳到一个通用的官网首页，80% 流失。

### 设计原则

- **一设备一页**：不搞官网首页、导航、关于我们、新闻动态
- **三屏讲完**：参数→案例→联系，不需要滚动超过三次
- **加载 2 秒内**：纯静态 HTML，不依赖后端/数据库

### 页面结构

```
┌─────────────────────────────────────┐
│          设备核心视频（自动播放静音）   │  ← 视频 + 设备名 + 一句话 USP
├─────────────────────────────────────┤
│  参数对比表（同类设备横比）            │  ← 参数 + 优势高亮
│  ┌─────────┬──────┬──────┐         │
│  │         │ LJ-800│ 同类  │         │
│  │ 线径    │ 0.1-1.2│ 0.2-0.8│      │
│  │ 转速    │ 3000  │ 2000  │         │
│  │ 精度    │±0.02 │ ±0.05 │         │
│  └─────────┴──────┴──────┘         │
├─────────────────────────────────────┤
│  客户案例 + 工厂实拍                   │  ← 案例图片 + 验厂视频片段
│  [照片] 深圳某无人机公司 × 3台         │
│  [照片] 台州某电机厂 × 6台             │
├─────────────────────────────────────┤
│  CTA 区                              │  ← 大按钮 + 电话 + 微信
│  [获取选型方案 →] [一键拨号] [加微信]   │
├─────────────────────────────────────┤
│  底部：公司全称 + 地址 + 备案号         │
└─────────────────────────────────────┘
```

### 技术实现

- 纯静态 HTML + 内联 CSS（一个文件，零依赖）
- 数据从 `taizhou_longjiang.json` + `product-geo.json` 注入
- 用 Python 模板引擎渲染（Jinja2 或直接 f-string 拼接）
- 输出文件：`landing/{设备ID}.html`
- 托管：nginx 静态目录 或 cpolar 映射

- 耗时：约 3 天

## 工期汇总

| 步骤 | 工期 | 优先级 | 难度 |
|------|------|--------|------|
| ① 发布层 | 2天 | P0（视频→四平台就是交付物的质变） | 低（纯 ffmpeg crop + LLM变体） |
| ② GEO 数据 | 1天 | P0（成本最低，效果最直接） | 低（JSON 生成+JSON-LD嵌入） |
| ③ 落地页 | 3天 | P1（需要案例素材，可先出模板） | 中（HTML模板+Jinja2渲染） |
| **合计** | **6天** | — | — |

---

## 执行次序

1. **先做 ② GEO**（1天）——不上线就能跑，Stage 2 加一个生成分支即可
2. **再做 ① 发布层**（2天）——四平台版本是交付物的核心差异化
3. **最后做 ③ 落地页**（3天）——依赖前两步的数据，案例素材可以和客户同步准备

## 验收标准

管线跑通后，输入一组绕线机素材，输出：

```
output/
├── stage3/final_h.mp4              # 横版成片
├── publish/
│   ├── douyin.mp4 + metadata.json  # 抖音版（竖版+标题+标签）
│   ├── kuaishou.mp4 + metadata.json
│   ├── shipinhao.mp4 + metadata.json
│   └── xiaohongshu.mp4 + metadata.json
├── geo/
│   └── lj800-product-geo.json      # GEO 结构化数据
└── landing/
    └── lj800.html                   # 落地页（加载 < 2s，含 JSON-LD）
```

工业设备采购决策周期长（7-90天），一条视频的价值不在播放量，在**6个月后客户搜"绕线机"时能不能找到你**。这三步解决的就是这个问题。
