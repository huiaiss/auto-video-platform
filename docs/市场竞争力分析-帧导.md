### 帧导的定位三角

| 维度 | Google Pomelli | Adobe Firefly | Vali | **帧导** |
|------|---------------|---------------|------|----------|
| 价格 | 免费 | $30/月 | 未公开(企业) | **按量计费, 100x-1000x 低于传统** |
| 输入 | 1张产品图 | 1-3张产品图 | 1张鞋图 | **8-10张多角度实拍** |
| 创意方式 | AI自动 | AI自动 | 模板+DB匹配 | **AI创意驱动, 每双鞋独一无二** |
| 场景来源 | AI生成(通用) | AI生成(通用) | 预设模板 | **根据产品属性定制场景描述→生成** |
| 输出 | 商品图 | 商品图 | 商品套图 | **商品套图(80+张) + 带货视频** |
| 平台适配 | 无 | 无 | 电商平台 | **六平台尺寸+四平台发布策略** |
| 目标用户 | 小卖家 | 设计师 | 鞋品牌 | **鞋厂(工厂直达)** |
| 闭环 | 出图 | 出图 | 出图 | **出图+出片+发布策略** |

### 核心差异化：三个"独一无二"

1. **AI创意驱动不是模板** — 每双鞋生成独立的创意概念、场景描述、配色方案、模特方向。老爹鞋="城市漫游者"，妈妈鞋="午后慢时光"，不是从20个模板里选一个。
2. **出图+出片一管打穿** — 同一套产品图输入，既出80+张静态套图，也出带货视频，不是两个独立工具。
3. **工厂直达，零中间环节** — 工厂寄鞋→拍8-10张→一键出图+出片，省去摄影棚/模特/设计师/剪辑师。

## 二、竞争格局：各家的牌和我们怎么赢

### A. 静态商品图赛道

| 竞品 | 优势 | 劣势 | 我们的应对 |
|------|------|------|-----------|
| **Google Pomelli Photoshoot** (2026) | 免费, Google生态, 上手快 | 泛品, 无鞋类理解, 场景通用化 | 鞋类垂直深度 Google做不到 |
| **Adobe Firefly Product Gen** | 品牌信任, 与PS/AE联动 | 需要设计技能, 单张生成 | 批量80+张, 零学习成本 |
| **Canva + NVIDIA** | 模板丰富, 有设计自由度 | 模板=千篇一律, 无AI创意 | AI定制场景 vs 手动选模板 |
| **Vali** (鞋类专用) | 200+鞋型DB, 91.3%首过率, 70%快发布周期 | 仅出图, 不出片 | **我们出图+出片, 创意不如我们** |
| **Youzan/WeChat电商工具** | 中国市场, 商家基数大 | 功能弱, 质量低 | 质量维度碾压 |

### B. AI视频赛道

| 竞品 | 优势 | 劣势 | 我们的应对 |
|------|------|------|-----------|
| **Seedance 2.0** (字节跳动) | 视频生成强, 生态好 | 不专门做商品视频, 无电商组件 | 电商专用组件(6组件库) |
| **PixVerse Shoplift** | AI视频生成 | 无商品分析, 无创意大脑 | 我们有引擎A创意大脑 |
| **Designkit** | 模板化 | 不是AI驱动 | AI凭产品属性决定内容 |
| **传统制作** ($500-10000/条) | 质量高, 定制 | 贵, 慢, 不可复制 | 100x-1000x成本优势 |

### C. 我们真正的护城河

```
第一层: 鞋类数据+创意知识库
  └── 每双鞋→品类识别(老爹鞋/玛丽珍/乐福鞋...)→风格映射→受众画像→场景描述
      这是通用平台永远不会做的垂直深耕

第二层: 创意引擎不是模板
  └── DeepSeek Vision 读图 + DeepSeek Chat 创意大纲 = 每双鞋独有创意
      Google/Adobe/Canva 都是模板化思维

第三层: 出图+出片一根管输出
  └── 没有竞品同时做这个。Vali 只出图, Seedance 只出片
      帧导是唯一"给你80张图+1条片"的平台
```

## 四、各端口质量标准

### 端口1: 产品分析输出 (引擎A)

| 指标 | 标准 | 检测方式 |
|------|------|---------|
| 品类准确率 | ≥90% | 人工抽检 + 鞋类知识库对比 |
| 风格标签 | ≥3个有效标签 | 与品类默认标签交叉验证 |
| 受众画像 | 年龄+性别+消费力 | DeepSeek 输出格式校验 |
| QA总分 | ≥70/100 | gatekeeper 7维评分 |
| 响应时间 | ≤30s | API 超时监控 |

### 端口2: 场景背景图 (引擎B)

| 指标 | 标准 | 检测方式 |
|------|------|---------|
| 分辨率 | ≥1600x1600px | Pillow 尺寸检测 |
| 风格匹配度 | CLIPScore ≥0.3 | CLIP 模型打分 |
| 色调一致性 | 与创意配色 ΔE<15 | 直方图对比 |
| 文件大小 | >1KB | 文件系统检查 |
| 生成成功率 | ≥95% | API 调用监控 |

### 端口3: 商品套图 (引擎C1) — 最关键的商业输出

| 指标 | 标准 | 依据 |
|------|------|------|
| 背景色 | RGB(255,255,255) ±5 | **Amazon强制标准** |
| 产品占比 | ≥85% 画面面积 | 电商平台最佳实践 |
| 最低分辨率 | 2000px 长边 | Amazon/Ebay 最低要求 |
| 产品完整性 | rembg 边缘无残缺 | 边缘像素检测 ≤2%异常 |
| CLIPScore | ≥0.3 (与产品图) | 低于0.3触发重新生成 |
| 文件格式 | PNG, 无压缩伪影 | 电商平台要求 |
| 平台尺寸 | 六平台精确匹配 | 各平台上传规范 |
| 输出数量 | 场景×角度×平台 ≥80张 | 产品规格承诺 |
| 一致性 | 同产品跨场景色调稳定 | 直方图批量对比 |

### 端口4: 带货视频 (引擎C2)

| 指标 | 标准 | 检测方式 |
|------|------|---------|
| 分辨率 | 1080x1920 (抖音竖屏) | FFprobe |
| 帧率 | 30fps | FFprobe |
| 时长 | 25-55s (抖音推荐) | FFprobe |
| 音频采样率 | 48000Hz | FFprobe |
| 音频响度 | -14 LUFS | pyloudnorm |
| TTS准确率 | 无错别字/漏读 | 脚本-SRT 对比 |
| GSAP时间轴 | 无丢帧/跳帧 | Chromium 渲染日志 |
| 组件完整性 | 6个电商组件全部渲染 | HTML 结构验证 |
| 品牌色 | CSS变量正确注入 | HTML 头部分析 |
| 字幕同步 | 偏差 ≤0.2s | SRT-Audio 对齐检测 |

## 六、结论：可以打

### 我们有的
- 鞋类垂直深度 (Google/Adobe 不会做)
- AI创意驱动不是模板 (核心差异化)
- 出图+出片一根管 (唯一)
- 工厂直达零环节 (效率)
- 全中文+国内平台适配 (本地化)

### 我们要补的
- C1出图管线整合 (P1, 2天)
- 质量自动检测体系 (每个端口植入质量门)
- 实际鞋类产品测试 (凯妮芬鞋做 test case)
- 种子用户验证 (工厂老板真实反馈)

### 市场时机
- 2025-2026 是 AI 商品内容爆发期
- Google/Adobe 入局验证了市场需求
- 但它们走泛品路线, 留下了垂直品类的空间
- 中国鞋业出口市场巨大 (莆田/温州/东莞), 但 AI 工具渗透率几乎为零
- **现在切入, 是正确的时间。**
