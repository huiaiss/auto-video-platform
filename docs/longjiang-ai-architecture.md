## 一、核心理念：域名即漏斗

霸首网用一个域名聚合了 **内容生产→搜索拦截→落地页→询盘** 整条链路。longjiang-ai.com 同样采用单域名聚合策略，将视频生产线、GEO 结构化数据、产品落地页全部汇聚在一个域名下，形成 AI 搜索引擎友好的询盘漏斗。

### 双语策略

```diff
+ 中文版 (longjiang-ai.com/)       → 国内买家：百度/抖音/DeepSeek/豆包
+ 英文版 (longjiang-ai.com/en/)     → 海外买家：Google/Bing/LinkedIn
+ 同一域名，不同语言路径，共享品牌信任
```

## 二、完整站点架构

### 2.1 URL 结构树

```
longjiang-ai.com
│
├─ /                                # 首页 — 品牌门户 + 设备导航 + 视频轮播
├─ /index.html                      # 同上（冗余入口）
│
├─ /products/                       # 产品总览页 — 5大品类分类导航
│   ├─ /products/LJ-9WD-1.html      # 37个GEO落地页（现有，统一改造）
│   ├─ /products/单工位飞叉外绕机.html
│   ├─ /products/...                # 共37个设备专属页
│
├─ /videos/                         # 视频展示页 — auto出片后自动生成
│   ├─ /videos/index.html           # 视频列表（网格布局，按设备分类）
│   └─ /videos/{设备ID}.html        # 单视频详情页（可选的）
│
├─ /cases/                          # 客户案例
│   ├─ /cases/index.html            # 案例列表
│   └─ /cases/{案例}.html           # 单个案例详情
│
├─ /contact/                        # 询盘页面
│   └─ /contact/index.html          # 表单 + 电话 + 微信 + 地址
│
├─ /about/                          # 公司介绍
│   └─ /about/index.html            # 工厂实拍 + 资质证书 + 团队
│
├─ /service/                        # 服务承诺
│   └─ /service/index.html          # 售前/售中/售后
│
├─ /sitemap.xml                     # AI爬虫入口（自动更新）
├─ /robots.txt                      # 允许所有AI爬虫
├─ /404.html                        # 自定义404页
│
└─ assets/                          # 静态资源
    ├─ /assets/css/common.css       # 全局样式
    ├─ /assets/js/common.js         # 全局JS（导航/页脚/CTA）
    └─ /assets/img/                 # 图片资产
```

### 2.2 对标霸首网漏斗模型

| 漏斗阶段 | 霸首网做法 | auto 做法 | longjiang-ai.com 对应 |
|----------|-----------|-----------|---------------------|
| 流量获取 | 人工追踪抖音热点 → 48h出片 → 矩阵发布 | Stage 1-3 自动出片 → 四平台发布层 | 视频内容 → 抖音/快手/视频号/小红书 → 引导至域名 |
| 搜索拦截 | 87%长尾词SEO + 百度首页 | GEO结构化数据 → AI搜索引擎优先推荐 | sitemap.xml + JSON-LD + Schema.org → DeepSeek/豆包/元宝 |
| 信任建立 | 工厂剧场化 + 方言共鸣 + 老板IP | 实拍素材 + 精度参数 + 39份PPT专业数据 | 首页视频轮播 + 产品落地页参数 + 案例/资质 |
| 转化成交 | 评论区置顶 → 私信 → 留资 | CTA按钮 → 电话/微信 → 询盘表单 | 每个页面的固定浮动CTA + 底部统一询盘区 |

### 2.3 AI爬虫友好策略（核心壁垒）

```
┌─ sitemap.xml ─────────────────────────────────────────────┐
│ 每次auto出片后自动更新，提交到：                             │
│   ├─ 百度搜索资源平台 (https://ziyuan.baidu.com)            │
│   ├─ Google Search Console                                 │
│   ├─ DeepSeek 爬虫 (DeepBot/1.0)                           │
│   └─ 豆包/TikTok 爬虫                                      │

│   ├─ <script type="application/ld+json"> 结构化数据         │
│   ├─ <meta name="description"> 精准摘要                    │
│   ├─ canonical URL 避免重复                                │
│   └─ Open Graph / Twitter Card 社交分享标签                 │
├─ /robots.txt ──────────────────────────────────────────────┤
│   User-agent: *                                            │
│   Allow: /                                                 │
│   Sitemap: https://longjiang-ai.com/sitemap.xml            │
└────────────────────────────────────────────────────────────┘
```

## 七、页面生成策略

| 页面类型 | 生成方式 | 更新频率 | 工具 |
|---------|---------|---------|------|
| 首页 `/index.html` | 手动设计 + 静态HTML | 按需 | 手写 |
| 产品落地页 37个 | 模板引擎批量生成（已存在） | 新增产品时 | `python build_landing.py` |
| 视频展示页 `/videos/` | Pipeline Stage 4 扩展 | 每次出片后 | `python build_video_page.py` |
| sitemap.xml | 自动爬取+生成 | 每次部署时 | `python build_sitemap.py` |
| 统一导航/页脚 | JS注入 或 模板批量改写 | 一次性 | `python patch_landing.py` |
| 案例页 | 手动添加 | 按需 | 手写 |
