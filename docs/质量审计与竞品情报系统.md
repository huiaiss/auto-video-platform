# 帧导 — 质量审计 & 竞品情报Agent系统

## 一、诚实审计：当前各端口实际产出质量

### 端口A：产品分析 + 创意大纲

| 维度 | 标准 | 实际 | 差距 |
|------|------|------|------|
| 品类识别准确率 | ≥90% | 实测可识别老爹鞋/玛丽珍/乐福鞋等主流品类 | ✅ 达标 |
| 风格标签 | ≥3个 | 通常输出3-6个 | ✅ 达标 |
| 材质识别 | 准确描述 | "网面透气纹理""哑光PU质感" — 描述到位 | ✅ 达标 |
| 受众画像 | 年龄+性别+场景 | "18-28女/日常通勤/校园/逛街" | ✅ 达标 |
| 色彩分析 | hex推测 | #F5F0E8等 — 有推测但无校验 | ⚠️ 推测无验证 |
| 创意概念 | 每产品独有 | "步履时光"概念+故事+场景 | ✅ 达标 |
| 场景描述 | 2-3个具体场景 | 窗边午后+小区路漫步 — 具体可用 | ✅ 达标 |
| 模特穿搭建议 | 具体服装搭配 | "浅色牛仔裤+白色T恤" — 具体 | ✅ 达标 |
| QA评分 | ≥70/100 | gatekeeper 7维检查存在 | ⚠️ 未见实际评分数据 |

**端口A结论：核心功能可用，缺少的是系统性的质量度量（没有跑过批量测试，没有统计过准确率数字）。**

### 端口B：内容工厂

| 维度 | 标准 | 实际 | 差距 |
|------|------|------|------|
| 场景图生成 | CLIPScore ≥0.3 | 程序化纹理可用，无AI生图 | ❌ 未接AI生图API |
| 口播脚本 | 7 beats + outro | pipeline_test_v8输出7 beat + outro, 32.5s | ✅ 达标 |
| TTS语音 | 可懂、自然 | edge-tts zh-CN-YunxiNeural 可用 | ✅ 达标 |
| 素材管线 | Beat→素材匹配 | asset_pipeline + Seedream 4.5补漏已跑通 | ✅ 达标 |
| SRT字幕 | 同步偏差≤0.2s | 可生成但未自动校验 | ⚠️ 无自动校验 |

**端口B结论：脚本+TTS+素材管线已通，场景图生成是最大短板（目前只有程序化纹理，没有真正的AI场景图）。**

### 端口C1：商品套图（出图）

| 维度 | 标准 | 实际 | 差距 |
|------|------|------|------|
| rembg抠图 | 边缘无残缺 | image_processor 在 visual-hub 有代码 | ❌ 未迁入帧导 |
| 场景合成 | 自然融合 | 代码存在但未串联到管线 | ❌ 未整合 |
| 平台尺寸 | 六平台精确 | platform_adapter 在 visual-hub | ❌ 未整合 |
| QA过滤 | 自动异常检测 | qa_filter 待迁入 | ❌ 未整合 |
| 批量输出 | 80+张 PNG | 完全未打通 | ❌ 最大短板 |

**端口C1结论：这是最大的缺口。代码都在 visual-hub，但从未在帧导管线里跑通过一次完整的"产品图→80张套图"流程。**

### 端口C2：带货视频（出片）

| 维度 | 标准 | 实际 | 差距 |
|------|------|------|------|

## 三、竞品情报Agent系统设计

### 核心思路

不能只做一次调研就完了。竞品在迭代（字节两周更新一次模型），价格在变动，新产品在冒出。需要一个**持续运转的竞品雷达**，在做项目决策时自动提供最新情报。

### 架构总览

```
                         ┌────────────────────────┐
                         │   竞品情报中枢          │
                         │   (Competitive Intel)   │
                         └───────────┬────────────┘
                                     │
          ┌──────────────┬───────────┼───────────┬──────────────┐
          ▼              ▼           ▼           ▼              ▼
    ┌──────────┐  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
    │爬取Agent │  │爬取Agent │ │爬取Agent │ │爬取Agent │ │爬取Agent │
    │ 即梦/即创 │  │通义/绘蛙 │ │ 潮际好麦  │ │ Vali     │ │DeskClaw  │
    └────┬─────┘  └────┬─────┘ └────┬─────┘ └────┬─────┘ └────┬─────┘
         │             │            │            │            │
         └─────────────┴────────────┴────────────┴────────────┘
                                     │
                                     ▼
                          ┌────────────────────┐
                          │   分析Agent         │
                          │  对比/差距/趋势     │
                          └────────┬───────────┘
                                   │
                                   ▼
                          ┌────────────────────┐
                          │   决策Agent         │
                          │  策略建议/预警      │
                          └────────────────────┘
```

### Agent 1: 竞品爬虫 (5个子Agent)

每个子Agent负责监控一个竞品：

```
Agent: 即梦/即创 Monitor
├── 爬取目标: 即梦官网 + 即创平台 + 抖音官方公告
├── 监控维度:
│   ├── 价格变动 (免费额度变化、会员价调整)
│   ├── 模型升级 (Seedream/Seedance版本号)
│   ├── 新功能上线 (鞋类模板、批量生成、电商专用)
│   ├── 输出质量抽样 (下载公开demo图/视频，记录分辨率/细节)
│   └── 用户评价 (社交媒体反馈、投诉、好评)
├── 频率: 每周全量 + 重大更新即时告警
└── 数据存储: SQLite competitive_intel.db

Agent: 通义万相/绘蛙 Monitor
├── 爬取目标: 阿里云官网 + 绘蛙文档 + 鹿班更新日志
├── 监控维度: 同上 + 淘宝电商生态绑定深度
└── 频率: 每两周

Agent: 潮际好麦 Monitor
├── 爬取目标: 潮际好麦官网 + 社交媒体 + 客户案例
├── 监控维度: 价格、客户数、品类扩展(是否进入鞋类)
└── 频率: 每两周

Agent: Vali Monitor (最关键鞋类竞品)
├── 爬取目标: valimart.net + 惠利玛公众号 + 红蜻蜓公开数据
├── 监控维度:
│   ├── 硬件价格变动
│   ├── AI拍照机新功能
│   ├── 鞋型数据库扩展
│   ├── 新客户案例
│   └── 鞋类AI设计平台更新
└── 频率: 每周

Agent: DeskClaw Monitor
├── 爬取目标: 36氪 + DeskClaw官网 + NoDesk AI动态
├── 监控维度: 融资、功能更新、定价、鞋类视频能力
└── 频率: 每两周
```

### Agent 2: 分析引擎

```
输入: 5个爬虫Agent的原始数据
处理:
  1. 价格对比矩阵: 各竞品单图/单视频成本 → 排名
  2. 功能差距分析: 竞品有新功能 → 标记我们缺失的
  3. 质量基准: 收集竞品公开demo → CLIPScore/分辨率/细节度对比
  4. 趋势预警: 检测到竞品战略转向(如潮际好麦开始做鞋) → 红色告警
  5. 机会发现: 竞品弱点/用户抱怨 → 我们可切入的点

输出: 每周竞品情报简报 (markdown)
```

### Agent 3: 决策引擎

```
输入: 分析引擎输出 + 帧导当前状态
输出:
  1. 优先级调整建议: "Vali降价了, 加快C1出图落地"
  2. 差异化强化: "即梦新增了鞋类模板, 我们要强化创意引擎的故事性"
  3. 定价参考: "市场均价0.5元/图, 我们的成本结构能支持什么价格"
  4. 合作机会: "DeskClaw用的是Seedance 2.0, 我们C2也可以接"
```

### 技术实现

### 数据存储结构

```sql
-- 竞品基础信息
CREATE TABLE competitors (
    id INTEGER PRIMARY KEY,
    name TEXT,           -- 即梦AI / 通义万相 / Vali ...
    category TEXT,       -- platform / vertical / hardware
    url TEXT,
    first_tracked_at TIMESTAMP
);

-- 价格变动记录
CREATE TABLE price_history (
    id INTEGER PRIMARY KEY,
    competitor_id INTEGER REFERENCES competitors(id),
    product TEXT,        -- 商品图生成 / 视频生成 / 会员
    price_per_unit REAL, -- 单价(元)
    currency TEXT,
    free_quota TEXT,     -- 免费额度描述
    recorded_at TIMESTAMP
);

-- 功能更新记录
CREATE TABLE feature_updates (
    id INTEGER PRIMARY KEY,
    competitor_id INTEGER REFERENCES competitors(id),
    feature_name TEXT,
    description TEXT,
    threat_level TEXT,   -- high / medium / low
    our_gap TEXT,        -- 我们是否有对应功能
    recorded_at TIMESTAMP
);

-- 质量采样
CREATE TABLE quality_samples (
    id INTEGER PRIMARY KEY,
    competitor_id INTEGER REFERENCES competitors(id),
    sample_type TEXT,    -- image / video
    sample_url TEXT,
    resolution TEXT,
    clip_score REAL,
    human_rating INTEGER, -- 1-5
    notes TEXT,
    recorded_at TIMESTAMP
);

-- 情报简报
CREATE TABLE intel_briefs (
    id INTEGER PRIMARY KEY,
    week_start DATE,
    summary TEXT,
    key_findings TEXT,
    recommendations TEXT,
    threat_level TEXT,
    created_at TIMESTAMP
);
```

## 七、告警触发规则

```
RED ALERT (立即通知):
├── 核心竞品(即梦/Vali)推出鞋类专用功能
├── 竞品价格骤降50%以上
├── 竞品获得重要融资
├── 竞品与鞋业大厂(SHEIN/安踏/红蜻蜓)达成合作
└── 新竞品出现且直接对标帧导

YELLOW WARNING (每周简报):
├── 竞品模型版本更新
├── 竞品UI/UX改版
├── 竞品客户数突破里程碑
└── 竞品新品类扩展

GREEN OPPORTUNITY (每月分析):
├── 竞品用户投诉/差评集中点
├── 竞品放弃的功能
├── 市场新需求未被满足
└── 跨品类机会
```
