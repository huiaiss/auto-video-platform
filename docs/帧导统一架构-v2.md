# 帧导 (FrameCraft) — 统一架构全景图

## 核心定位 & 突破口

```
工厂寄鞋 → 拍8-10张多角度图 → 帧导自动:
  ① 识别品类/风格/受众/材质
  ② 生成品牌创意大纲 + 场景描述
  ③ 生成场景背景图
  ④ 批量合成 80+ 张商品套图 (多场景×多角度×多平台尺寸)
  ⑤ 生成带货视频 (TTS口播 × 电商动画组件 × Chromium渲染 → MP4)
```

**突破口**: "AI 读图 → 定制场景 → 一键出图+出片" — 不是通用模板，是每双鞋独一无二的创意产出。

--------------------------------------------------------------------------

## 项目合并决策

| 来源 | 角色 | 资产 |
|------|------|------|
| **auto-video-platform** (帧导) | 统一项目主体 | 渲染管线、组件库、Web UI |
| **visual-hub** | 创意大脑 + API 层 | 产品分析、创意大纲、场景生成、QA |

**合并后**: visual-hub 的 services/ 迁移到帧导，api/ 合并到帧导 web/，
两个项目不再独立维护。

## 数据流全景

```
POST /api/upload  (8-10张产品图)
        │
        ▼
  [引擎A: 创意大脑]
  classify_category()    → "老爹鞋 / 复古厚底 / 街头风 / 18-28女"
  analyze_product()      → 材质/质感/受众画像
  generate_creative_brief() → "城市漫游者" 概念 / 3场景 / 配色
  review_all()           → QA 评分 85/100 ✓
        │
        ▼
  [引擎B: 内容工厂]
  generate_scenes_from_brief() → 5张场景背景图
  generate_script()      → Script (7 beats + outro)
  build_tts()            → narration.mp3 + subtitles.srt
  resolve_assets()       → beat→image 匹配 + AI生图补漏
        │
        ├──────────────────────────┐
        ▼                          ▼
  [引擎C1: 出图]            [引擎C2: 出片]
  for each scene:           for each beat:
    rembg(product)            pick_component(beat)
    composite(bg, product)    component.html/css/gsap
    resize(platform)          CompositionBuilder.assemble()
    qa_filter(image)          ChromiumRenderer.render()
        │                          │
        ▼                          ▼
  output/images/             output/videos/
    product_场景1_taobao.png   index.html → output.mp4
    product_场景1_douyin.png
    ... (80+ files)
```
